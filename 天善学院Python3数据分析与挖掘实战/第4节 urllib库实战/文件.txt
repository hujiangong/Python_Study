<div id="article_content" class="article_content tracking-ad" data-mod=popu_307  data-dsm = "post" >
        <div class="markdown_views"><p>今天在更新一条记录时，突然发现自己竟然连update时嵌套使用select都不会用。摸索着写了一条，还报错了。类似如下</p>



<pre class="prettyprint"><code class=" hljs sql"><span class="hljs-operator"><span class="hljs-keyword">CREATE</span> <span class="hljs-keyword">TABLE</span> t3(
num1 <span class="hljs-keyword">SMALLINT</span>,
num2 <span class="hljs-keyword">SMALLINT</span>
);</span></code></pre>



<pre class="prettyprint"><code class=" hljs sql"><span class="hljs-operator"><span class="hljs-keyword">INSERT</span> <span class="hljs-keyword">INTO</span> t3 <span class="hljs-keyword">VALUES</span>(<span class="hljs-number">20</span>,<span class="hljs-number">10</span>),(<span class="hljs-number">30</span>,<span class="hljs-number">40</span>);</span></code></pre>

<table>
<thead>
<tr>
  <th align="center">num1</th>
  <th align="center">num2</th>
</tr>
</thead>
<tbody><tr>
  <td align="center">20</td>
  <td align="center">10</td>
</tr>
<tr>
  <td align="center">30</td>
  <td align="center">40</td>
</tr>
</tbody></table>




<pre class="prettyprint"><code class=" hljs sql"><span class="hljs-operator"><span class="hljs-keyword">UPDATE</span> t3 <span class="hljs-keyword">SET</span> num1=(
<span class="hljs-keyword">SELECT</span> num2 <span class="hljs-keyword">FROM</span> t3 <span class="hljs-keyword">WHERE</span> num1=<span class="hljs-number">30</span>
) <span class="hljs-keyword">WHERE</span> num2=<span class="hljs-number">10</span></span></code></pre>

<p>报错如下，大意是不能在FROM字句中使用要更新的目标表’t3’</p>

<pre class="prettyprint"><code class=" hljs vhdl">[Err] <span class="hljs-number">1093</span> - You can<span class="hljs-attribute">'t</span> specify target table <span class="hljs-attribute">'t3</span>' <span class="hljs-keyword">for</span> update <span class="hljs-keyword">in</span> FROM clause</code></pre>

<p>解决办法：(来源：<a href="http://blog.csdn.net/afeiqiang/article/details/8589535">http://blog.csdn.net/afeiqiang/article/details/8589535</a>) <br>
既然MySQL是通过临时表来实现FROM子句里面的嵌套查询，那么把嵌套查询装进另外一个嵌套查询里，可使FROM子句查询和保存都是在临时表里进行，然后间接地在外围查询被引用。 <br>
以下语句执行正确</p>

<pre class="prettyprint"><code class=" hljs sql"><span class="hljs-operator"><span class="hljs-keyword">UPDATE</span> t3 <span class="hljs-keyword">SET</span> num1=(
<span class="hljs-keyword">SELECT</span> num2 <span class="hljs-keyword">FROM</span>
(<span class="hljs-keyword">SELECT</span> * <span class="hljs-keyword">FROM</span> t3 <span class="hljs-keyword">WHERE</span> num1=<span class="hljs-number">30</span>) temp
) <span class="hljs-keyword">WHERE</span> num2=<span class="hljs-number">10</span></span></code></pre></div>
